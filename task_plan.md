# Task Plan: 写字课动画工坊

## Goal
从零重建一个面向小学语文教师的单页工具：输入一个汉字后，可查看规范笔顺演示，并将动画导出用于课件。

## Scope
- 重写应用界面与源代码，不沿用旧页面逻辑。
- 使用 Vite、原生 JavaScript 和 Hanzi Writer。
- 提供单字输入、固定米字格笔顺播放/重播和课堂视频导出。

## Phases
| Phase | Status | Notes |
| --- | --- | --- |
| 1. 明确产品流程与新文件边界 | complete | 根据补充需求移除非必要功能 |
| 2. 重建界面和交互状态 | complete | 固定米字格，教师备课场景，桌面优先 |
| 3. 完成动画与视频导出 | complete | 视频根据浏览器能力导出 MP4 或 WebM |
| 4. 构建并浏览器验收 | complete | 使用未缓存生字“鼎”验证本地字库生成成功 |

## Errors Encountered
| Error | Resolution |
| --- | --- |
| 当前目录不是 Git 仓库，无法用 `git status` 查看改动 | 直接以工作区文件为准进行重建 |
| 清理功能时首个补丁匹配了错误的按钮文本 | 读取准确片段后，按当前文件内容完成定向修改 |
| 沙箱禁止 Node / pnpm 解析 `C:\Users\test` 路径 | Node 语法检查改在授权后的沙箱外完成；安装与构建同样需外部执行 |
| 点击生成可能因外部 CDN 无法访问而失败 | 将 `hanzi-writer-data` 随应用打包，动画改从本地读取 |
| 导出视频在部分播放器中只播放到“春”的前五笔 | 排查为 MediaRecorder WebM 时长元数据不可靠，正在按真实录制时长修正容器 |
| 从配置的 npm 镜像安装 `fix-webm-duration` 时连接被中断 | 改用 npm 官方注册源成功获取同一依赖 |
| 浏览器验收环境无法向本地页面键入中文 | 完成生产构建与录制代码校验；请以新导出“春”视频作最终播放确认 |
| PowerShell 通过标准输入传入中文文件名时被转换为 `?` | 改用 Unicode 码点拼接文件名进行时长分析 |
| 尝试加入 `mp4-muxer` 时发现包已弃用且锁文件重命名失败 | 改用该项目推荐的现行替代库 `mediabunny` 实现 MP4 输出 |
| 临时验收入口首次打开到既有应用服务而非测试页面 | 使用独立空闲端口并核对测试页标题后重试 |
| 用户说明视频可完整播放，只有文件封面显示半成字 | 将完整成字停留延长至 7.5 秒，使中段缩略图抽帧落在完整字画面 |
| PPT 默认封面显示视频起始空白米字格，且长尾停留导致视频过长 | 将完整成字作为首帧停留 1 秒，随后播放笔顺，并将末尾停留收回 1 秒 |
| 本地验收页被浏览器安全策略明确阻止打开 | 不绕过限制；以导出时序实现与正式构建验证为准 |
| 输入框允许确认两个汉字，但中文拼音输入不能被固定字符长度干扰 | 移除 HTML 长度截断，在 `compositionend`/普通输入后仅保留首个有效汉字 |
